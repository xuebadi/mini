# Internationalization (i18n)

TinyWorld's gameplay UI is localized into **English (`en`), French (`fr`), Simplified
Chinese (`zh`), and Spanish (`es`)**. This document is the architecture + reference. For
the day-to-day "add a string / translate it" loop, use the **`tinyworld-i18n` skill**
(`.claude/skills/tinyworld-i18n/SKILL.md`).

## Why this design

TinyWorld is a static, single-page app assembled from classic `<script>` modules with no
bundler, and it must run from `file://` as well as a server. That rules out fetching JSON
locale files at runtime (CORS / `file://` failures). So:

- **Locale data ships as JS**, not JSON. Each `engine/i18n/<code>.js` is a tiny IIFE that
  registers its dictionary onto `window.TWI18N_DATA.<code>`. `publish.sh` already copies
  all of `engine/` recursively, so locale files deploy with no build change.
- **All engine modules share one global scope** (they're classic scripts; the 2-space
  indentation is cosmetic from the old god-file split). So a `t()` defined in a script
  loaded first is callable as bare `t('key')` everywhere.

## Files

| File | Role |
|---|---|
| `engine/i18n/en.js` | **Authoritative** keyâ†’string map. Defines the full key set. |
| `engine/i18n/fr.js` `es.js` `zh.js` | Translations. Must match `en.js`'s keys exactly. |
| `engine/i18n/i18n-core.js` | Locale resolution, `t()`, `tx()`, `has()`, `apply()`, `setLocale()`. |
| `tools/i18n-check.js` | Parity + key-usage checker. Runs inside `npm run check` / `publish.sh`. |
| `tiny-world-builder.html` | Loads the locale scripts + core before the engine modules; carries `data-i18n*` attributes. |

Load order in the HTML (right after `vendor/tinyworld-auth.js`):

```html
<script src="engine/i18n/en.js"></script>
<script src="engine/i18n/fr.js"></script>
<script src="engine/i18n/es.js"></script>
<script src="engine/i18n/zh.js"></script>
<script src="engine/i18n/i18n-core.js"></script>
<script src="engine/world/00-prelude.js"></script>
...
```

Locale data must load **before** the core (so the dictionaries exist when it resolves the
locale), and the core must load **before** the engine modules (so their top-level code can
translate at build time).

## Runtime API (all global)

- `t(key, params?)` â€” active locale â†’ English fallback â†’ the key itself (so a missing
  string is visible, never blank). `params` interpolates `{name}` placeholders.
- `tx(key, fallback)` â€” translation if the key exists, else the supplied `fallback`. Used
  to localize hard-coded labels in data arrays while keeping their English default.
- `has(key)` â€” true if a real translation exists (active or English).
- `TWI18N.apply(root?)` â€” translates `data-i18n*` attributes under `root` (default
  `document`). Runs once on `DOMContentLoaded`.
- `TWI18N.setLocale(code)` â€” persists the choice and reloads.
- `TWI18N.locale`, `TWI18N.supported`, `TWI18N.names` (endonyms).

### HTML attribute hooks (handled by `apply()`)

| Attribute | Sets |
|---|---|
| `data-i18n` | `textContent` |
| `data-i18n-title` | `title` |
| `data-i18n-tooltip` | `data-tooltip` (the custom tooltip system) |
| `data-i18n-placeholder` | `placeholder` |
| `data-i18n-aria-label` | `aria-label` |

Put `data-i18n` only on **leaf** elements. For an element with child elements (e.g. a
`<label>` around a `<select>`), wrap the text in a `<span data-i18n="â€¦">` so `apply()`
doesn't replace the children.

## Language selection

Resolution order, computed once at boot:

1. `?lang=<code>` URL parameter (also persisted, so it sticks afterward)
2. `localStorage['tinyworld:lang']`
3. `navigator.language(s)` â€” region-tagged codes match their base (`fr-CA` â†’ `fr`)
4. English

The in-app switcher is **Settings â†’ Workspace â†’ Language** (`#ui-lang-mode`). Changing it
calls `setLocale()`, which persists the choice and **reloads the page** ("reload-on-switch").
Everything re-renders correctly from a clean boot â€” no per-surface re-render wiring. The
in-progress home grid survives because it autosaves to `localStorage['tinyworld:v1']`
(`engine/world/29-persistence-api.js`) and is restored on load.

## Scope

Translated: toolbar/tools/variants, world menu, camera/view, time & weather, crowd panel,
sound, layers/properties, welcome/how-to, top-bar + control tooltips, settings tab names +
Workspace section, the mode HUD, the radial menu, player-facing toasts.

Intentionally English: account/auth, AI/agent panel, API keys, developer overlay,
credits/sponsors, token ticker, and the dense technical settings sliders (Rendering /
Materials / Environment / Crowd-advanced / AI).

## Commands

```bash
npm run i18n:report   # per-locale coverage + any unknown keys
npm run i18n:check    # fails on missing/extra/empty keys or unknown used keys
npm run check         # full publish gate (includes i18n:check)
npm run dev           # then open ?lang=fr | ?lang=zh | ?lang=es to verify
```
