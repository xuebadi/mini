## Findings considered and rejected

- **Event listener leak (initWelcomeDialog)**: `initWelcomeDialog` is called exactly once; no accumulation occurs. Not a real leak.
- **CSRF via missing Origin header**: app uses `Authorization: Bearer` tokens (not cookies), making browser CSRF inapplicable. `sameOriginWriteGuard` returning `true` for missing-Origin is intentional for non-browser API clients.
- **Cloud sync async race**: the `twCloudWorldSyncing` boolean flag + `twCloudWorldSyncPending` drain is a correct single-threaded mutex. JavaScript's event loop prevents re-entrance between async yields.
- **Fade material UUID cache collision**: Three.js assigns a unique UUID per Material instance on creation; two instances cannot share a UUID. False positive.
